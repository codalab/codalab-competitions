Create a Competition

CodaLab competitions provide an opportunity for researchers, developers and algorithmists to create solutions for problems across a wide range of domains, and advance the state of the art for their respective areas of interest. This guide covers everything you need to know to create and manage a CodaLab competition, from uploading data to evaluating competition results.

Prerequisites

Required:

  • Windows Azure™ storage account (requires Azure subscription). If you do not already have an Azure account, you will need to create one. See Configure Azure for detailed instructions.
  • AzCopy

Optional:

  • PowerShell (installed by default on Windows 7 and newer)
  • Windows Azure PowerShell. Read the Azure PowerShell documentation.

Competition Overview

Competitions are conducted in two phases: training and competition. The appropriate data is made available to participants at each phase of the competition. During the training phase, participants have access to training data to refine their algorithms. During the competition phase, participants are provided with competition data to generate results which they can then submit to the competition.

The files that make up a competition are as follows:

  • Participant Instructions
  • Reference Data
  • Evaluation Program
  • Manifest Files

Participant Instructions

The competition organizer is responsible for providing participants with documentation that describes the goals of the competition, input and output file formats, and details about how submissions will be evaluated and ranked. By default each competition has the following pages:

  • Overview - A brief description of the competition.
  • Evaluation - Specify how results will be evaluated.
  • Terms and Conditions - Describe terms that participants must agree to.
  • Get Data - Provide competition data to participants.

The instructions should be formatted in HTML and then added when running the Create Competition Wizard.

Reference Data

Data forms the basis for competitions. In the training phase, participants use a set of training data to create their algorithms. In the competition phase a different set of data (testing) is used to evaluate and rank algorithms. Reference data can take many different forms. In our example competition the reference data consists of a set of CT scan images to evaluate (.mhd), and a corresponding set of coordinates for each image (.lml).

Evaluation Program

The evaluation program compares the submitted results with the answers and passes the scores to the CodaLab leaderboards.

Manifest File

The manifest file (competition.yaml) is used to define all aspects of the competition.

Note: There are no .zip archives in the example competition as downloaded. This is so that the files can be properly downloaded. You will need to manually create .zip archives from the dataset, program, and reference folders.

Create a Manifest File

The manifest file (competition.yaml) defines all aspects of the competition, and is defined using YAML. The following example shows the structure of competition.yaml:

# Build an example competition
---
title: Example Competition
description: This is a competition to test the competition bundle system. It should be able to create a competition from this bundle.
image: logo.jpg
has_registration: True
end_date: 2013-12-31
html: 
    overview: overview.html
    evaluation: evaluation.html
    terms: terms_and_conditions.html
    data: data.html
phases:
    1:
        phasenumber: 1
        label: "Test"
        start_date: 2013-06-30
        max_submissions: 100
        scoring_program: program.zip
        reference_data: reference.zip
        datasets: 
            1:
                name: Data 1
                url: http://spreadsheets.google.com/pub?key=pyj6tScZqmEfbZyl0qjbiRQ&output=xls
                description: Example Dataset
            2:
                name: Data 2
                url: http://spreadsheets.google.com/pub?key=0AgogXXPMARyldGJqTDRfNHBWODJMRWlZaVhNclhNZXc&output=xls 
                description: Example Dataset
    2:
        phasenumber: 2
        label: "Training"
        start_date: 2013-08-30
        max_submissions: 3
        scoring_program: program.zip
        reference_data: reference.zip
    3:
        phasenumber: 3
        label: "Challenge"
        start_date: 2013-09-30
        max_submissions: 3
        scoring_program: program.zip
        reference_data: reference.zip
        datasets: 
            1:
                name: Challenge Data
                url: http://spreadsheets.google.com/pub?key=t9GL1nIZdtxszJbjKErN2Hg&output=xls
                description: Example challenge data
leaderboard:
    leaderboards:
        RESULTS: &RESULTS
            label: Results
            rank: 1
    columns:
        Difference:
            group: *RESULTS
            label: Difference
            rank: 1

See Building a Competition Bundle for details about each entry in the manifest file.

Add Folder Contents to ZIP Archives

1. Add the contents of the program, reference, and dataset data folders to individual ZIP archives. Using our example competition files, zip the contents of the following folders: 

C:\MyCompetition\dataset1 -> dataset1.zip
C:\MyCompetition\dataset2 -> dataset2.zip
C:\MyCompetition\dataset3 -> dataset3.zip
C:\MyCompetition\dataset4 -> dataset4.zip
C:\MyCompetition\reference -> reference.zip
C:\MyCompetition\program -> program.zip

2. Create a .zip archive of C:\MyCompetition. This will be known as the "competition bundle".

Create a CodaLab Competition

In this step you'll use the Create Competition Wizard to create a competition on CodaLab.

To Create a Competition
1. Log in to CodaLab.
2. Click My CodaLab.
3. Click Competitions I'm Running
4. Click Create New Competition.
5. Click Choose file and select the .zip archive containing the competition bundle. 6. Click Submit to upload the package data and create the competition.
