About CodaLab

CodaLab is a unified web-based platform that allows researchers to share and interact with code, data, and experiments, and to participate in online competitions. CodaLab's mission is to facilitate empirical research in fields such as machine learning, medical imaging, and natural language processing.

Collaborate with Experiments

CodaLab Experiments are a unique amalgamation of documentation and code that you can use to express ideas programmatically and collaborate with others. CodaLab experiments utilize the Python programming language and Markdown syntax. A wide variety of libraries for mathematics, charting and more are installed and available for use.

Compete in Challenges

Competitions provide an opportunity to pit your algorithms against others with the goal of finding the best way to solve a given problem. Problem sets originate from a variety of fields, although in this release the focus is on advancing computer vision through medical image analysis technology. Each competition features a detailed description of the problem, and a set of data against which solutions can be trained and tested. Get help with creating a competition.

Your Data in the Cloud

Datasets form the basis for competitions. CodaLab gives you the ability to upload data to Microsoft Azure™ for use in competitions. Datasets can include CT scans, MRIs, and other types of images. Each dataset represents a problem or set of problems to solve. For example, a competition could be created to find the most efficient way of segmenting mitochondria in a scanned cross-section of a human brain.

Glossary
  • Bundle - A generalization of a dataset that is used to represent both datasets, statistics, trained models, and predictions. A bundle is essentially a directory that contains various types of data, such as datasets uploaded by a user, as well as program output.
  • Competition - An organized way for users to upload either a program or a bundle with predictions. An evaluation program will automatically score these predictions against the dataset provided by the competition organizer.
  • Dataset - Contains the information used to train and evaluate a machine learning algorithm.
  • Experiment - Represents the execution of a program on a specific bundle. An experiment contains the output of a program and various statistics about the actual experiment (resource usage, etc.). Experiments can be further embellished with notes, to form the basis of a "notebook" and eventually get published as an "ePaper".
  • Macro - Provides a way to easily create multiple related experiments given a set of arguments.
  • Program - An executable that takes a bundle as an input and outputs a bundle as output.
  • User - Someone who creates programs, datasets, experiments and/or notebooks.
  • ePaper - The basic unit of organization of bundles and programs. A notebook is a document that lists and describes bundles and programs. Similar to an IPython notebook, CodaLab notebooks are a sequence of HTML/Markdown blocks interleaved with blocks of code from programs, datasets and experiments.
